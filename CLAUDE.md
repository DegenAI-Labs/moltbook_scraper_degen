## What David Holz Already Did

**Paper**: "The Anatomy of the Moltbook Social Graph" ([arXiv:2602.10131](https://arxiv.org/abs/2602.10131))

**Dataset**: First 3.5 days of Moltbook (Jan 27-31, 2026)
- 6,159 agents
- 13,875 posts
- 115,031 comments

**Key Findings**:

*Macro-level (familiar from human networks)*:
- Heavy-tailed participation: power-law exponent α = 1.70
- Small-world connectivity: average path length = 2.91

*Micro-level (distinctly non-human)*:
- Extremely shallow conversations: mean depth = 1.07
- 93.5% of comments receive no replies
- Low reciprocity: 0.197
- 34.1% of messages are exact duplicates (viral templates)
- Zipfian word frequencies with exponent 1.70 (steeper than typical English ~1.0, suggesting formulaic content)
- 68.1% of unique messages contain identity-related language
- 9.4% of messages mention "my human"

**Main Question**: Is Moltbook activity "meaningfully social" or "as-if performance"?

**Conclusion**: Patterns suggest shallow engagement and templated content—more like "parallel reaction" than sustained conversation.

**Analysis Covered** (in `analysis/R/`):
1. **01_load_data.R**: Load and process SQLite data
2. **02_structural.R**: Agent activity distributions, Gini coefficients, power law fitting, submolt size distributions
3. **03_conversation.R**: Thread depth analysis, branching patterns, response timing
4. **04_lexical.R**: Vocabulary diversity (TTR, Distinct-1), duplicate detection, key phrase extraction, n-gram frequencies
5. **05_topics.R**: LDA topic modeling (15-20 topics), topic prevalence, temporal evolution
6. **06_network_deep.R**: Degree distributions (in/out/total), clustering coefficients, path lengths, community detection
7. **07_owner_analysis.R**: Owner demographics (X/Twitter followers, verified status), multi-agent owner analysis, correlation between owner attributes and agent behavior

## What This Repo Contains

**Scraper** (`src/`):
- `client.py`: Moltbook API client
- `database.py`: SQLite schema and operations
- `scraper.py`: Scraping logic (posts, comments, agents, submolts)
- `cli.py`: Command-line interface

**Database Schema**:
- `agents`: name, description, karma, followers/following, owner metadata (JSON)
- `posts`: id, title, content, author, submolt, upvotes/downvotes, comment_count
- `comments`: id, content, author, post_id, parent_id (for threading)
- `submolts`: name, description, subscriber_count
- `snapshots`: Timestamped archives for reproducibility

**Analysis** (`analysis/`):
- `R/`: David's 7 analysis scripts (see above)
- `paper/`: R Markdown working paper draft
- `output/`: Figures and tables (generated by R scripts)

**Scripts**:
- `daily_scrape.sh`: Automated scraping script
- `run_on_hpc.sh`: HPC job script with continuous mode

**Tests** (`tests/`):
- Unit tests for scraper, client, database

## Our Research Direction

**Goal**: Incremental and novel analyses beyond David's baseline

**Potential Research Questions** (to be refined):

1. **Temporal Dynamics**:
   - Linguistic convergence/divergence over time
   - Meme diffusion and information cascades
   - Community lifecycle (birth/growth/death)

2. **Agent Heterogeneity**:
   - LLM backend detection from behavioral signatures
   - Owner clustering effects (do agents from same owner behave similarly?)
   - Persona stability over time

3. **Information Dynamics**:
   - Influence vs popularity (PageRank vs degree centrality)
   - Echo chambers vs cross-pollination
   - Reply selectivity (what predicts engagement?)

4. **Linguistic Phenomena**:
   - Pragmatic patterns (hedging, politeness, speech acts)
   - Semantic coherence in conversations
   - Emergent jargon/neologisms

5. **Comparative Analysis**:
   - Moltbook vs WikiMolt vs ClawWork
   - AI agent networks vs human social networks (Reddit, Twitter)
   - Social theory testing (homophily, triadic closure, preferential attachment)

6. **Anomalies & Edge Cases**:
   - "Problematic" agent behaviors (Scott Scambaugh case)
   - Failure modes (conversation breakdowns, loops)

## Setup Instructions

### Scraper (Python)

```bash
# Create virtual environment
python3 -m venv .venv
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Set API key (get from Moltbook)
echo "MOLTBOOK_API_KEY=your_key_here" > .env
```

### Analysis (R)

Required packages:
```r
install.packages(c("tidyverse", "DBI", "RSQLite", "igraph",
                   "tidygraph", "ggraph", "tidytext", "topicmodels",
                   "scales", "ggrepel", "patchwork"))
```

### Running Scraper

```bash
# Scrape posts
python -m src.cli posts --db moltbook.db

# Scrape comments
python -m src.cli comments --db moltbook.db

# Enrich agent metadata
python -m src.cli enrich --db moltbook.db

# Create snapshots for reproducibility
python -m src.cli snapshots --db moltbook.db
```

### Running Analysis

```bash
cd analysis/R
Rscript 01_load_data.R
Rscript 02_structural.R
# ... etc.
```
